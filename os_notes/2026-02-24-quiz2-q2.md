## Study Guide: xv6 Kernel and Testing Framework

## gradelib.py — Testing Framework

### Core Test Execution
```python
@test(points, title=None)
def register_test(fn):
    # Decorator to register test functions
    # Tracks TOTAL (earned) vs POSSIBLE (available) points
    # Captures test output and handles failures
```

**Key Functions:**
- `run_tests()` — Execute registered tests with optional filtering
- `assert_equal(got, expect, msg)` — Compare values with detailed error output
- `assert_lines_match(text, *regexps, no=[])` — Verify output matches regex patterns

### QEMU Control
The `Runner`, `QEMU`, and `GDBClient` classes manage kernel testing:
- `run_qemu(*monitors)` — Boot kernel with optional breakpoints/callbacks
- `user_test(binary, *monitors)` — Run user-level program in QEMU
- `match(*regexps)` — Assert QEMU output matches patterns

> **💡 Hint:** Monitors are callbacks that process QEMU output in real-time to detect test completion or trigger breakpoints.

## Key xv6 Components

### Buffer Cache (kernel/bio.c)

| Function | Purpose |
|----------|---------|
| `bread(dev, blockno)` | Get cached block, read if needed |
| `bwrite(buf)` | Write block to disk |
| `brelse(buf)` | Release block, update LRU list |
| `bpin/bunpin(buf)` | Prevent/allow eviction |

**LRU Strategy:** `bcache.head.next` = most recent; `bcache.head.prev` = least recent

**Critical Section:** Hold `bcache.lock` when modifying refcnt; hold `b->lock` (sleeplock) when reading/writing data.

---

### File System Structure (kernel/fs.c, kernel/fs.h)

**Disk Layout:**
```
[boot] [superblock] [log] [inode blocks] [bitmap] [data blocks]
```

**Key Structures:**
- `inode` — In-memory file metadata (ref count, type, size, block addresses)
- `dirent` — Directory entry (inode number, name)
- `superblock` — FS metadata (size, block count, inode count, log location)

**Important:** Inodes use NDIRECT (12) direct pointers + 1 indirect pointer for up to 268 total blocks.

---

### Process Management (kernel/proc.c, kernel/proc.h)

**Process State Machine:**
```
UNUSED → USED → RUNNABLE → RUNNING → SLEEPING → RUNNABLE → ZOMBIE → UNUSED
```

**Key Fields:**
- `pagetable` — User page table (3-level Sv39)
- `trapframe` — Saved CPU state for trap handling
- `context` — Kernel stack frame for scheduling
- `ofile[NOFILE]` — Open file descriptors
- `cwd` — Current working directory inode

**Synchronization:**
- `pid_lock` — Protects PID allocation
- `wait_lock` — Coordinates parent-child waits (acquire before `p->lock`)
- `p->lock` — Protects process state

---

### Virtual Memory (kernel/vm.c)

**Page Table Walking:**
```c
pte_t *walk(pagetable_t pagetable, uint64 va, int alloc);
// Returns PTE pointer; creates intermediate pages if alloc==1
```

**Key Macros:**
- `PX(level, va)` — Extract 9-bit index at level (0=leaf, 1=mid, 2=top)
- `PA2PTE(pa)` — Convert physical address to PTE format
- `PTE2PA(pte)` — Extract physical address from PTE

**Guard Pages:** Stack guard at `TRAPFRAME - PGSIZE` to catch overflow.

---

### Locking Primitives

| Type | Use Case | Properties |
|------|----------|-----------|
| `spinlock` | Short critical sections, interrupt handlers | Disables interrupts, busy-wait |
| `sleeplock` | Long-term locks (inode, buffer) | Yields CPU, re-acquires spinlock |

**Deadlock Prevention:** Acquire locks in consistent order (e.g., always `itable.lock` before `ip->lock`).

---

### System Calls

**Read/Write Path:**
```
sys_read() → fileread() → readi() [inode lock] → bread() [buffer]
```

**File Descriptor Table:**
- Per-process array `ofile[0..NOFILE-1]`
- `ofile[0]` = stdin, `ofile[1]` = stdout, `ofile[2]` = stderr
- Reference counted (`filedup`, `fileclose`)

---

### Pipes (kernel/pipe.c)

**Structure:**
```c
struct pipe {
  char data[PIPESIZE];  // 512 bytes
  uint nread, nwrite;   // byte counters (never reset)
  int readopen, writeopen;
};
```

**Semantics:**
- Write blocks if full; wakes readers
- Read blocks if empty and writer open; returns 0 if writer closed
- Both threads call `wakeup()` after state changes to avoid lost wakeups

---

### Disk I/O (kernel/virtio_disk.c)

**Virtio Queue:** Descriptor ring, available ring (driver→device), used ring (device→driver)

**Operation:**
1. Allocate 3 descriptors: [request header] → [data buffer] → [status byte]
2. Add to avail ring; notify device
3. Sleep until device writes to used ring and calls interrupt handler

---

### Traps & Exceptions (kernel/trap.c)

**User → Kernel:**
1. Exception/interrupt → `usertrap()`
2. Dispatch on `scause` (ecall=8, external=9, timer=5)
3. Handle syscall, return via `usertrapret()`

**Restore User State:**
```c
w_sepc(p->trapframe->epc);      // User PC
w_sstatus(x & ~SSTATUS_SPP);    // User mode
// Jump to trampoline code to restore registers and sret
```

## Concurrency Patterns

### Safe Counter Update
```c
acquire(&lock);
counter++;
release(&lock);
```

### Producer-Consumer (Pipes)
```c
acquire(&pi->lock);
while(pi->nwrite == pi->nread + PIPESIZE)  // buffer full
  sleep(&pi->nwrite, &pi->lock);            // release lock, wake on nwrite
// write data
wakeup(&pi->nread);  // signal reader
release(&pi->lock);
```

### Inode Modification
```c
ilock(ip);        // acquire sleeplock
ip->size += n;    // modify
iupdate(ip);      // write to disk (via log_write)
iunlock(ip);
```

> **⚠️ Warning:** Never hold spinlock during sleep. Always use sleeplock for I/O operations.

---

## Common Pitfalls

| Mistake | Consequence | Fix |
|---------|-------------|-----|
| Sleep while holding spinlock | Deadlock | Use sleeplock for long ops |
| Lost wakeup | Thread never wakes | Call `wakeup()` after every state change |
| Forget iput() | Inode leak | Use `iunlockput()` for cleanup |
| Race in refcount | Use-after-free | Always acquire lock before checking/modifying |
| Page table leak | Memory exhaustion | Call `uvmfree()` in error paths |
