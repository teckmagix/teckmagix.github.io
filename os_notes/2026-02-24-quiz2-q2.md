## xv6 Kernel & User Space Code Study Guide

## Buffer Cache (bio.c)

### Key Structures & Functions

**Buffer Cache Structure:**
- LRU (Least Recently Used) doubly-linked list maintains block order
- `bcache.head.next` = most recent, `bcache.head.prev` = least recent
- Spinlock protects the cache, sleeplock protects individual buffers
- Reference counting (`refcnt`) tracks active users

| Function | Purpose |
|----------|---------|
| `binit()` | Initialize empty buffer cache |
| `bread()` | Get buffer with disk block data |
| `bwrite()` | Write buffer back to disk |
| `brelse()` | Release buffer, move to MRU position |
| `bget()` | Find cached block or allocate new buffer |

### Core Workflow
1. **bread()**: Call `bget()` → if not valid, read from disk
2. **bget()**: Search cache for block → if found, increment refcnt → else allocate LRU unused buffer
3. **brelse()**: Release sleeplock → decrement refcnt → if refcnt==0, move to MRU head

> **💡 Hint:** Two-level locking: spinlock for cache metadata, sleeplock for buffer data access. This prevents deadlock and allows concurrent operations.

## Console & Input Handling (console.c)

### Input/Output Flow

**Console Structure:**
- Circular buffer: `cons.buf[INPUT_BUF_SIZE]`
- `cons.r` = read index, `cons.w` = write index, `cons.e` = edit index

| Handler | Purpose |
|---------|---------|
| `consoleintr()` | Handle input interrupts (^P, ^U, ^H, ^D) |
| `consoleread()` | User read syscall: wait for line input |
| `consolewrite()` | User write syscall: output characters |

### Special Characters
- **^D (EOF)**: Marks end-of-input
- **^P**: Print process list
- **^U**: Kill entire line
- **^H / DEL**: Backspace

> **⚠️ Warning:** `consoleread()` blocks until newline or EOF. Lock ordering: acquire cons.lock before sleep to avoid losing wakeups.

## File System Layers

### Layer 1: Block Allocation (fs.c)

**Disk Layout:**
```
[boot | superblock | log | inode blocks | bitmap | data blocks]
```

| Function | Purpose |
|----------|---------|
| `balloc()` | Allocate free disk block |
| `bfree()` | Free disk block, clear bitmap bit |
| `readsb()` | Read superblock |

### Layer 2: Inode Management (fs.c)

**Inode Structure:**
- **In-memory**: includes refcnt, validity, lock status
- **On-disk**: type, major, minor, nlink, size, addrs[NDIRECT+1]
- **Direct blocks**: `addrs[0..11]` point to data blocks
- **Indirect blocks**: `addrs[12]` points to block containing more addresses

| Function | Purpose |
|----------|---------|
| `iget()` | Find/create inode table entry |
| `ilock()` | Lock + read inode from disk if needed |
| `iput()` | Decrement refcnt, free if nlink==0 |
| `bmap()` | Get block number for file offset |

> **💡 Hint:** `iget()`/`iput()` manage refcnt for in-memory copies. `ilock()` ensures inode data is valid before use.

### Layer 3: File Operations (file.c)

| Function | Purpose |
|----------|---------|
| `fileread()` | Read from file descriptor |
| `filewrite()` | Write to file descriptor |
| `filedup()` | Duplicate file reference |
| `fileclose()` | Close and cleanup |

### Layer 4: Path Lookup & Directories (fs.c)

| Function | Purpose |
|----------|---------|
| `namei()` | Lookup path, return inode |
| `nameiparent()` | Lookup parent directory |
| `dirlookup()` | Find directory entry by name |
| `dirlink()` | Add new directory entry |

## Process Management (proc.c)

### Process Structure
```c
struct proc {
  pagetable_t pagetable;    // User page table
  uint64 sz;                // Process size in bytes
  struct file *ofile[16];   // Open file descriptors
  struct inode *cwd;        // Current working directory
  enum procstate state;     // UNUSED, USED, SLEEPING, RUNNABLE, RUNNING, ZOMBIE
}
```

### Process Lifecycle

| State | Meaning |
|-------|---------|
| UNUSED | Slot free in process table |
| USED | Allocated but not runnable |
| SLEEPING | Blocked on sleep() |
| RUNNABLE | Ready to run |
| RUNNING | Currently executing |
| ZOMBIE | Exited, waiting for parent wait() |

### Key Functions

| Function | Purpose |
|----------|---------|
| `fork()` | Create child process copy |
| `exec()` | Replace process with new program |
| `exit()` | Terminate, reparent children |
| `wait()` | Parent waits for child exit |
| `scheduler()` | CPU scheduler loop |
| `sched()` | Switch to scheduler |

> **💡 Hint:** `fork()` copies memory and file descriptors. `exec()` replaces memory but keeps file descriptors and environment.

## Memory Management (vm.c, kalloc.c)

### Kernel Memory Allocator (kalloc.c)

- **Units**: 4096-byte pages only
- **Structure**: Linked list of free pages
- `kalloc()` → returns page or NULL
- `kfree()` → return page to freelist

### Virtual Memory (vm.c)

**Page Table Levels**: 3 (Sv39 RISC-V scheme)
- Level 2, 1, 0: Each 9-bit index into 512-entry table
- Leaf PTE contains physical address

| Function | Purpose |
|----------|---------|
| `walk()` | Navigate page table, allocate if needed |
| `mappages()` | Create PTEs for virtual range |
| `uvmalloc()` | Allocate user memory pages |
| `uvmdealloc()` | Free user memory pages |
| `uvmcopy()` | Copy parent page table to child |

## Synchronization Primitives

### Spinlock (spinlock.c)

- **Busy-waiting** lock using atomic swap
- Disables interrupts (`push_off()`) to prevent deadlock
- **Use when**: Critical section is short

| Function | Purpose |
|----------|---------|
| `acquire()` | Spin until lock acquired |
| `release()` | Release and re-enable interrupts |

### Sleep Lock (sleeplock.c)

- **Yields CPU** while waiting
- Protects long-term operations
- Built on spinlock + sleep/wakeup
- **Use when**: Lock held for extended time

| Function | Purpose |
|----------|---------|
| `acquiresleep()` | Wait for lock (may sleep) |
| `releasesleep()` | Release and wake waiters |

## Logging & Crash Recovery (log.c)

### Transaction Model

**Write-ahead logging:**
1. Modify buffer cache blocks
2. Call `log_write()` to record in-memory
3. On commit: write all logged blocks to disk log
4. Write header (atomic point)
5. Install blocks at home locations
6. Clear log

| Function | Purpose |
|----------|---------|
| `begin_op()` | Start FS operation |
| `end_op()` | End operation, commit if last |
| `log_write()` | Mark buffer for logging |

> **💡 Hint:** Log header write is atomic commit point. Crash recovery replays committed transactions.

## Pipes (pipe.c)

**Structure:**
```c
struct pipe {
  struct spinlock lock;
  char data[PIPESIZE];      // Circular buffer
  uint nread, nwrite;        // Byte counts
  int readopen, writeopen;   // Open FD count
}
```

**Operations:**
- `piperead()`: Wait until data available or write end closed
- `pipewrite()`: Write bytes, wait if buffer full
- `pipeclose()`: Decrement open count, free when both closed

## Execution & System Calls (exec.c, syscall.c)

### exec() Workflow

1. Read ELF header, validate magic
2. Allocate new page table
3. Load program segments (LOAD type)
4. Set up user stack with arguments
5. Switch to new page table
6. Jump to entry point

### Syscall Dispatch

- `syscall()` reads syscall number from `a7` register
- Looks up handler in `syscalls[]` array
- Calls handler, stores return in `a0`

## Traps & Interrupts (trap.c)

### Trap Types

| scause | Type | Handler |
|--------|------|---------|
| 8 | Syscall | `syscall()` |
| 9 (external) | Device IRQ | `devintr()` |
| 0x8000...1 | Timer | `clockintr()` |

### Flow

1. **usertrap()**: User mode trap handler
   - Save registers to trapframe
   - Enable interrupts for syscalls
   - Dispatch to appropriate handler

2. **usertrapret()**: Return to user space
   - Disable interrupts
   - Load user registers from trapframe
   - Execute `sret` to return to user mode

## Device Drivers

### UART (uart.c)

- **Transmit**: Buffered output with interrupt-driven refill
- **Receive**: Line-buffered input via `consoleintr()`

### Virtio Disk (virtio_disk.c)

- **DMA descriptors**: Request/response buffers
- **Split operations**: 3-descriptor chains (header, data, status)
- **Interrupt-driven**: I/O completion via interrupt handler

## Key Design Patterns

### Two-Phase Locking
- Acquire all locks in consistent order to prevent deadlock
- Example: inode operations lock parent before child

### Reference Counting
- Inodes: `refcnt` tracks in-memory copies in use
- Files: `ref` tracks open file descriptors
- Prevents premature deallocation

### LRU Eviction
- Buffer cache: evict least-recently-used block when full
- Improves cache hit rate for sequential access

### Transaction Semantics
- `begin_op()` / `end_op()` brackets atomic FS operations
- Log ensures all-or-nothing semantics across crashes
